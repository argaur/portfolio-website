Personal Homelab

A self-hosted Ubuntu 22.04 VM on Oracle Cloud free tier — running personal bots, automation scripts, and scheduled services 24/7 at zero cost.

Linux Oracle Cloud systemd Ubuntu 22.04
Setup

The Server

The homelab runs on an Oracle Cloud always-free tier VM — Ubuntu 22.04 LTS. Oracle Cloud's free tier provides a persistent VM with a public IP at no cost, making it suitable as a personal server for automation and bots that need to run continuously.

Oracle Cloud Always Free tier VM
Ubuntu 22.04 LTS operating system
SSH key auth no password login
What Runs on It

Services

The VM acts as the execution layer for personal automation projects. Each service runs as a systemd unit, ensuring automatic restart on failure and clean process management.

Active Services

  • Telegram Personal Assistant Bot — single listener process, keyword routing, always-on
  • LinkedIn Post Automation — systemd + cron job for scheduled LinkedIn publishing pipeline

Why systemd over a process manager like PM2? systemd is already on the OS, handles dependencies between services, integrates with the system journal for logging, and survives VM reboots without any additional setup.

Process Management Pattern

  • Long-running listeners (Telegram bot) → systemd service with Restart=always
  • Scheduled jobs (LinkedIn automation) → cron or systemd timer
  • One-off scripts → run manually over SSH
Access

Remote Access

The VM is accessed exclusively via SSH with key-based authentication. Password login is disabled. The SSH key is stored locally and used directly from the terminal.

SSH key auth removes an entire class of brute-force attack surface. Combined with Oracle Cloud's default network security rules, the VM is accessible only over SSH on the standard port.
