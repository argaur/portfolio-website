Development
Per-slice: Red (failing tests) → Green → Refactor → Instrument → Document → Review → Smoke-run → Commit. Between slices: 5-line PROGRESS.md + /compact.

Testing
Layer 0: Security pass (IDOR, authz, input sanitization). Layer 1: Unit + Integration + E2E. Layer 2: 8-category edge case checklist. Layer 3: Acceptance criteria.

Stages
S0
Data Model
All entities, relationships, every state (empty / loading / error / populated). Mandatory before any visual work. Output: DATA_MODEL.md.
S1
Brainstorm Named Reader added
1. Load context. 2. Fetch 2–3 live reference UIs. 3. Name 3 design risks (→ Stage 5 checklist).
4. Named Reader — one specific real person this product must land for. Name + role + context + sophistication in one sentence. Stored in Brand Guide.
S2
Interview
One question at a time. Step 0: Purpose Statement. Track A: design language derived from purpose (visual feel, references, fonts, negative constraint "what would make this look AI-generated?"). Track B: per-panel intent with real copy.
S3
Wireframes Friction Annotation added
Layout only. Component map + key screens + unhappy states (empty/error) + end-to-end flow.
Friction annotation: annotate each transition in the flow with one word — confusion / hesitation / exit / scroll / load / skip. One word per arrow. Surfaces UX risk at layout cost.
S4
Stitch — Design System + Brand Guide
Every token carries a value + usage rule. Component showcase. Inevitability test on the full set. Output: tailwind.config + CSS vars + components.json + brand-guide.md.
S5
Spec Doc
What changed this session and why. Required: Analytics Surface · Constraints Contract · Implementation cost flags · Design risk resolution (all 3 from S1) · Visual changelog with thumbnails.
S6
Handoff → Blueprint Phase 3
Design ends at spec sign-off. Packet: tailwind.config + CSS vars + components.json + shadcn install list + component showcase + signed SPEC.md.
