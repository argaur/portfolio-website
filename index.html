How I Think

Product Principles

  1. 01 Purpose before aesthetics — define what the artifact must prove before any visual choice
  2. 02 Systems thinking precedes solution design — understand the whole before optimising a part
  3. 03 Complexity is a product failure — if users need training, the design failed first
  4. 04 Build for the constraint, not the ideal — constraints reveal the real design problem
  5. 05 Medium is the message — the artifact itself should demonstrate the skill, not just claim it
  6. 06 Ship to learn — a hypothesis is only as good as the data that refutes or confirms it
