Product Principles

The manifesto for strategic logic — 7 principles, one capstone

01

Purpose Before Aesthetics

Define what the artifact must prove before any visual choice is made. Form follows function — always.

02

Systems Thinking First

Understand the whole before optimising any part. Everything is a node in a larger network — design for the ripple, not the splash.

03

Complexity Is Failure

If users need training, the design failed first. Strip away noise until only the essential remains. Simplicity is the hardest discipline.

04

Build For the Constraint

Constraints reveal the real design problem, not the ideal. Use them, don't fight them. The constraint is the brief.

05

Medium Is the Message

The artifact itself should demonstrate the skill, not just claim it. Every surface is a proof point. This portfolio is a case in point.

06

Ship to Learn

A hypothesis is only as good as the data that refutes or confirms it. The first version is a question. Iterate relentlessly.

07

Capstone Principle

The Inevitability Test

If the design language could be transplanted to a different product without feeling wrong, it was imposed, not derived. Every choice must be earned. Every pixel is a decision. Every interaction is an act of will.
