Personal Project

Telegram Personal Assistant Bot

An always-on personal automation bot running 24/7 on Oracle Cloud VM — keyword routing dispatches commands to modular handlers without a framework.

Python Telegram Bot API Oracle Cloud systemd
Overview

What It Is

A personal automation bot that runs continuously on an Oracle Cloud VM. It listens to incoming Telegram messages and dispatches them to different handlers based on keywords in the message. No framework, no third-party workflow engine — just a Python listener and a routing table.

The goal was a single, low-friction interface for personal automation tasks: capture an idea, log something to a note, trigger a script — all from Telegram without opening a browser or another app.

Architecture

How It Works

A single listener process runs as a systemd service on the VM. When a message arrives, the listener checks for registered keyword prefixes and routes to the appropriate handler module.

Design choice: One listener, many handlers. This keeps the entry point stable while making it easy to add or remove capabilities without touching the core routing logic.

Routing Rules

  • #idea — captures the message and saves it to the Notion Post Ideas database
  • #capture — saves the message to the Obsidian vault (Google Drive sync)
  • Additional routes are registered as new handlers are built

Infrastructure

24/7 uptime via systemd service
Oracle Cloud free tier VM (Ubuntu 22.04)
cron scheduled tasks alongside listener
Integrations

Connected Systems

The bot is part of a broader personal automation stack. It connects to the tools I already use daily, so capturing or triggering something requires only a single Telegram message.

  • Notion: Post Ideas DB, Posts DB, Saved Links DB — structured capture for LinkedIn pipeline
  • Obsidian Vault: synced via Google Drive, used for unstructured notes and project pages

Bot flow ideas and routing decisions are tracked in a dedicated Notion page to keep the routing table from growing uncontrolled.
