Section 01

Product Overview

Group travel planning is a $168.7 billion global market growing at 7.2% CAGR — faster than solo travel — yet the coordination layer that helps groups decide, plan, and execute trips together remains almost entirely unsolved. Today, groups stitch together 4–5 disconnected tools and one person absorbs the entire coordination burden.

This platform is an AI-native group trip coordination PWA — from the first 'let's go!' message to final expense settlement — making it safe and frictionless to align on preferences before planning begins.

Section 06

AI Layer Design

Overview

The AI layer has 4 distinct features across V1–V3. All use Claude API (claude-sonnet-4-6). Each has a defined trigger, input, prompt structure, output format, and fallback.

AI Feature 1 — Preference Synthesis V1 / P0

Trigger: All / majority of members complete preference survey

Input: Aggregated anonymized survey responses (budget ranges, dietary types, trip styles, constraints)

Claude Prompt:

"You are a group travel coordinator. Given these anonymous group preferences:
Budget ranges: {ranges}
Dietary types: {dietary}
Trip styles: {styles}
Constraints: {constraints}

Summarize: what does the group agree on, flag significant gaps, suggest 2-3 broad trip directions.
Be warm, concise, practical. Output: paragraph summary + JSON with keys:
budget_overlap_range, dietary_union, style_distribution, gap_flags[]"

Output: Natural language paragraph + structured JSON displayed as visual summary cards

Fallback: Rule-based aggregation (average budget, union of dietary constraints, mode of trip style) if Claude unavailable

AI Feature 2 — Destination Suggestions V2 / P1

Trigger: Organizer clicks "Get AI destination suggestions" after preference synthesis

Input: Preference synthesis output + trip type + group size + date range + origin city

Claude Prompt:

"Based on this group's preferences: {synthesis_summary}
Suggest 3 destination options for a {size}-person {trip_type} trip from {city} for {duration}.
For each provide: name, why_it_fits_this_group, estimated_cost_per_person_inr,
travel_time_from_origin, top_3_highlights[], watch_out_for.
Format as JSON array."

Output: 3 destination cards rendered on Destination Voting page — each with vote buttons

AI Feature 3 — Location Helper (GPS-aware) V3 / P2

Trigger: Active during trip (GPS permission granted); also proactive based on location changes

Architecture: Claude with tool use (claude-sonnet-4-6 tool_use beta)

Tools available to Claude:
- get_weather(lat, lng)                        → current conditions + 24h forecast
- search_nearby_eateries(lat, lng, dietary[])  → filtered by group dietary constraints
- get_road_conditions(route_polyline)          → crowd-sourced alerts, landslide flags
- find_nearest(lat, lng, category)             → ATM / petrol / medical / restroom
- search_local_forums(query, location)         → travel forum snippets for ground truth

Safety alerts: If trip tagged 'Kid Friendly' or 'Female Group' AND location matches risk area DB → send Web Push notification immediately

Personalization: Uses anonymous individual preference profiles from Trip Optimizer feature

AI Feature 4 — Trip Optimizer (Individual Suggestions) V3 / P2

Origin: Tab 2 idea from PRD

Concept: Each member has an anonymous individual preference profile (separate from group survey). Claude generates personalized "you might enjoy" suggestions for their sub-group time slots.

Profile inputs (anonymous form, per-person, not shared with group):
- Solo interests: food / culture / adventure / shopping / photography / nature
- Energy level: slow-paced / moderate / high-energy
- Budget flexibility: fixed / flexible
- Hidden gem vs popular: prefers off-beat or iconic?

Suggestions: solo experiences, local finds, divergent activities
that match individual (not group) preferences.
Shown only to that individual member.

Privacy: Individual suggestions never visible to other members. Profile stored anonymously (hashed user_id).

Section 07

API Design

All endpoints prefixed with /api/v1. FastAPI auto-generates OpenAPI docs at /docs. Auth via Auth.js session cookie on all protected routes.

Auth

POST/auth/google→ OAuth callback; create/update user; return session cookie
POST/auth/phone→ Initiate WhatsApp OTP for phone number
POST/auth/verify-otp→ Verify OTP; create Auth.js session

Trips

POST/trips→ Create trip (name, destination?, dates?, trip_type, group_size_estimate)
GET/trips→ List user's trips (active + archived)
GET/trips/{trip_id}→ Full trip detail
PUT/trips/{trip_id}→ Update trip details
DELETE/trips/{trip_id}→ Archive trip (soft delete)

Members

POST/trips/{trip_id}/invite→ Generate invite token + WhatsApp deep link
POST/trips/{trip_id}/join→ Join via invite token
GET/trips/{trip_id}/members→ List members with roles and join status

Preferences

POST/trips/{trip_id}/preferences→ Submit anonymous preference survey
GET/trips/{trip_id}/preferences/summary→ Aggregated anonymized group summary
GET/trips/{trip_id}/preferences/ai-synthesis→ Claude-generated natural language synthesis

Tasks

POST/trips/{trip_id}/tasks→ Create task (title, category, assigned_to, due_date)
GET/trips/{trip_id}/tasks→ List tasks (filter: status, assignee, category)
PUT/trips/{trip_id}/tasks/{task_id}→ Update status / reassign
POST/trips/{trip_id}/tasks/{task_id}/nudge→ Send push notification to assignee

Itinerary

GET/trips/{trip_id}/itinerary→ Full itinerary (all days + items)
POST/trips/{trip_id}/itinerary/days/{day}/items→ Add itinerary item to a day
PUT/trips/{trip_id}/itinerary/days/{day}/items/{item_id}→ Update item
POST/trips/{trip_id}/itinerary/ai-suggest→ Claude generates day activity suggestions

Votes

POST/trips/{trip_id}/votes→ Cast vote (vote_type, option_id, value: up|neutral|down)
GET/trips/{trip_id}/votes/{vote_type}→ Get vote tally for a vote type
POST/trips/{trip_id}/vote-options/ai-generate→ Claude generates destination vote options

Expenses

POST/trips/{trip_id}/expenses→ Log expense (amount, category, paid_by, split_type)
GET/trips/{trip_id}/expenses→ List expenses (filter: category, date)
GET/trips/{trip_id}/expenses/summary→ Per-person totals
GET/trips/{trip_id}/expenses/settlement→ Net balances + Razorpay UPI payload

Realtime

GET/trips/{trip_id}/stream→ SSE stream — yields JSON events for dashboard live updates

AI / Location

POST/ai/location-helper→ Claude location query (lat, lng, query, trip_context)
POST/ai/trip-optimizer→ Individual suggestions (trip_id, anonymous_profile)
